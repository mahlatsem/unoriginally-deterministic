3 lenses

A downstream consumer can source-extract this pair cleanly: every {path.to.token} reference in both files resolves, all three frontmatter sources exist, PRD/PRFAQ citations are verbatim-accurate, and the three invented sections carry the product's actual load. Shape fit and flow coverage are strong. The spine's honesty instincts are real and structural — theme inheritance instead of a brand palette, the state-machine perspective rejected as fabrication, unparsed-as-warning-not-error, the GT Miller-columns rejection with cited evidence.

Two lenses converge on the same underlying failure, and it is the one worth acting on: the spine's picture is more resolved than the extraction can prove, in exactly the three places COBOL gets hard. Dynamic CALL identifier targets, unparsed regions inside a rendered structure, and unexpanded copybooks each produce a view that looks uniformly verified while parts of it are inferred. That is the fabrication class this product exists to refuse, arriving through the back door. Separately, the accessibility floor names the right outcomes but commits no mechanism — for a node-edge canvas the mechanism is the hard part, and as written the screen-reader commitments are not implementable.

Two scope-level issues sit above the craft findings: the spine supersedes PRD OQ-3 and materially expands FR-5 with the flag recorded only in .memlog.md, so a consumer reading PRD + spines sees an unflagged contradiction with no declared winner; and "committed fully to v1" contains roughly five renderer-scale builds for a solo pre-gate maintainer, with no phasing against the PRD's blocking reliability gate. Neither requires redesign — the perspective model survives all three reviews intact. The claims, the sequencing, and the accessibility mechanisms need work.

Adversarial review

review-adversarial.md

Verdict: the spine's honesty instincts are real, but it fails its own discipline exactly where COBOL gets hard — dynamic CALLs, copybooks, and holes-inside-structures all render as more-resolved than extraction can prove — and it commits a solo pre-gate builder to roughly five renderer-scale projects with no phasing. Fixable without redesign: the perspective model survives; the claims and sequencing need cutting. 2 critical · 4 high · 7 medium · 2 low.

critical

Flow 3 demonstrates a capability the spine declares out of v1

EXPERIENCE.md:219–227 vs :147
Nomsa's molded question includes "show me where the Java invokes it," and step 4 says the view draws it. v1 extraction is COBOL + toolkit-TS only, so no molded tool over v1 primitives can answer it — and the spine's own Boundary paragraph forbids exactly this. The flagship molding flow delivers the overpromise the Boundary paragraph exists to prevent.
Fix: Rewrite the question to a COBOL-only gap, or have the flow show the honest refusal — COBOL-side reach drawn, Java boundary marked as edge-of-map. Drop "~20 lines."
critical

Dynamic CALL targets: the call graph is presented as resolved

EXPERIENCE.md:134–136
COBOL has CALL 'PROG' (static) and CALL WS-PROG-NAME (dynamic, target is a runtime value). The spine specifies CALL edges, sequence messages and transitive reach as if every target were statically known. Drawing a resolved edge for a dynamic CALL is a guess; omitting it is a silent hole — both forbidden. "In source order, as written" covers order, not target identity. Nothing in DESIGN provides a vocabulary for this: the parse succeeded, the target simply isn't static.
Fix: Add a third honesty state — unresolved target: dashed edge to a distinct ‹dynamic: WS-PROG-NAME› terminal node, own legend entry, popover showing the CALL line. Reach past a dynamic CALL states truncation. Ban heuristic target resolution.
high

Edges and order across unparsed holes imply unverified continuity

EXPERIENCE.md:135, 137, 138
Unparsed nodes are specified thoroughly; edges and order across a hole are not. A flowchart drawing A → ‹unparsed› → B asserts fall-through the parser never confirmed. So "mechanically derivable for any unit" is false precisely when a hole is present. The view renders uniformly, and the uniformity is the dishonesty.
Fix: Hole-adjacent edges render in the unparsed treatment; never draw an edge around a hole; sequence and call-flow show a marked gap in the order.
high

Full v1 commitment pre-gate hides five renderer-scale builds

EXPERIENCE.md:120; PRD §1, SM-2
Deterministic graph layout for 214 files with collapse and a layout toggle; a bespoke sequence renderer with guard and loop fragments; bidirectional editor sync; module-reach reads/writes (needs data-item mapping through copybooks — the heaviest extraction in the bundle, sitting in one table row); trail restore across re-derives. None is needed to run the reliability gate, which needs extraction, provenance primitives, one view, click-to-source.
Fix: Name phases inside the spine. Phase A (gate-sufficient): dependency map + flowchart, popover, jump-to-source, unparsed rendering. Phase B: sequence, reach, call-flow, trail, rail, Code sync, molded registration. Keep the model fully committed so Phase B slots in without redesign.
high

Copybook expansion is load-bearing and completely unspecified

EXPERIENCE.md:126–139
COPY is textual inclusion, often with REPLACING; tree-sitter parses as written. A paragraph PERFORMed in one file but defined in a copybook resolves nowhere. Worse, under REPLACING the provenance anchor for a replaced identifier points at a line that doesn't contain the name shown on the node — the click-through lands somewhere that doesn't say what the node says, breaking the core trust gesture.
Fix: State the model: v1 renders as-written; cross-COPY references become edges to the copybook node with both anchors; REPLACING noted plainly in the popover so click-through never surprises.
high

The provenance tick over-signals, and collides with unparsed nodes

DESIGN.md:187; EXPERIENCE.md:67, 113
DESIGN says the tick rides every element and its absence is impossible; the unparsed-node spec is silent on it — and a green "verified" mark on a region labelled unread conflates "we located it" with "we read it." Separately, the tick guarantees a span exists, not that it supports the drawn claim: a molded tool can emit a wrong edge whose provenance is a real but irrelevant line, rendering with starter-grade visual trust.
Fix: State that the tick means "carries a source span" — nothing more. Purge "verified to source" for "traces to PAYROLL.cbl:214". When the active lens is molded, the header names it with one-click "view analyzer source."

Accessibility review

review-a11y.md

Target floor: WCAG 2.2 AA as it applies to a VS Code webview extension, plus two platform realities — webviews receive raw key events, and theme tokens carry no contrast contract. The spine's accessibility intent is unusually strong for a graph product, but five of its seven bullets name outcomes with no committed mechanism, and for a node-edge canvas the mechanism is the hard part. 2 critical · 4 high · 8 medium · 2 low.

critical

No accessible-structure model or linearized alternative for the canvas

EXPERIENCE.md:110–116, 95
The spine commits to "every node/edge reachable" but never says what the accessibility tree contains; a rendered canvas exposes nothing by default and edges have no native announceable representation. The one linear surface offered — the provenance rail — lists source anchors, not topology, so a screen-reader user can audit provenance but never comprehend the view. That inverts the product's purpose.
Fix: Commit an explicit ARIA model (canvas as one tab stop, roving tabindex, nodes and edges focusable with name, kind and relationships) plus a per-perspective linearized equivalent — a Structure view sibling to the rail. This doubles as the 1.4.10 reflow answer.
critical

Meaning-bearing diagram semantics have no non-visual representation

EXPERIENCE.md:135–138, 64
Guards, cycles, GO TO distinction and read/write/call kinds are specified purely as visual marks. "Never color alone" names shape and legend — still visual-only. A screen-reader Nomsa cannot learn the TAXCALC call is guarded by IF RUN-MODE = 'YR-END' — the literal climax of Flow 2.
Fix: Every meaning-bearing mark gets a text equivalent in its accessible name — "calls TAXCALC, conditional: IF RUN-MODE = 'YR-END'", "GO TO (unstructured jump)", "writes EMPFILE", "decision block, lines 214–220".
high

Tab consumed by the canvas is a trap and orphans the chrome

EXPERIENCE.md:95 vs 116
If Tab walks nodes and edges, a 214-file graph puts hundreds of stops between the user and the perspective tabs, breadcrumb and rail toggle — directly contradicting the "ordinary tab-stops" claim.
Fix: Canvas is one tab stop; arrows plus type-ahead move within it; Tab exits to the next control. State the full focus ring including the Code split.
high

The trace outcome — the core trust action — is visual-only

EXPERIENCE.md:64, 94, 203
The #1 trust check is specified as paired lighting of element and editor line, with no announcement or focus outcome. The spine's own line — "a verifiability guarantee that only works with a pointer isn't a guarantee" — applies equally to vision.
Fix: Jump-to-source moves focus into the editor at the line; in-canvas trace fires a live-region status; the linked state is exposed as an aria state.
high

Theme inheritance does not guarantee contrast for this product's compositions

DESIGN.md:23–33, 80–92
Delegating to the theme inherits its choices, not a contrast contract. charts-* carries no guarantee against the widget background; editorWarning-foreground as label text is tuned for squiggles and frequently under 4.5:1 — exactly on the honest hole whose legibility is a brand commitment; the tick is a meaning-bearing graphic with no minimum size; the base edge stroke is a deliberately-muted token at 1px on a dense canvas.
Fix: Add a contrast-floor clause verifying each composed pair at render time with in-system fallbacks (swap to foreground, add a contrastBorder outline, thicken the mark); give the tick a minimum size.
high

Keybinding portability and conflicts

EXPERIENCE.md:94, 97, 99
Webviews receive raw DOM key events, so VS Code keybindings don't apply and remapped bindings aren't honored without explicit work. Alt+← is not Go Back on macOS, breaking the muscle-memory rationale on one of three named platforms. Esc collides with NVDA/JAWS focus-mode conventions, where it may drill out a grain as a destructive surprise.
Fix: Every primitive is also a contributable, remappable, palette-visible command; document macOS defaults; announce each Esc stage.
medium

review-a11y.md M1–M8
DESIGN says the trace highlight fades, EXPERIENCE says it persists (EXPERIENCE is correct) · popover "dismisses on moving away" fails 1.4.13 · Esc stages and trail restores unannounced · "high-contrast honored automatically" is an overclaim (HC removes fills) · no live-region strategy, so SR users get the picture without the header's caveats · Code-split sync can steal focus · no zoom, reflow, legibility-floor or reduced-motion commitments · molded tools inherit no accessibility contract.
