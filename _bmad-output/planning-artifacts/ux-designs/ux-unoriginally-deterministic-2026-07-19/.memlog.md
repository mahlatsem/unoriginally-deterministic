- (decision) Load-bearing UX invariants derived from PRD/PRFAQ (spines win on conflict): (1) Provenance is a first-class interaction — every drawn node/edge is click-to-source to the exact line; no source, no element. (2) Fail-loud honesty — tree-sitter ERROR nodes render as visibly-marked 'unparsed' regions, never omitted, never fabricated. (3) Deterministic/reproducible — no AI at run time, no per-run uncertainty, same code=same picture; no spinners implying nondeterminism. (4) Zero-friction start — starter views run on install with no account, no config, no tokens. (5) Molding happens in the engineer's own AI assistant OUTSIDE the toolkit UI; the toolkit surface is the extension-contract/primitives read as code + docs.

- (decision) Perspectives model CONFIRMED, committed FULLY to v1: one scope x switchable perspectives across three grains. System grain (node=program/module): dependency map, sequence diagram, module-reach tracer. Component grain (node=section/paragraph/function): call-flow. Unit grain (inside one paragraph/function): flowchart (control-flow) + code view. Drill = Enter/double-click down a grain, Esc/breadcrumb up; F12 = jump to source at every grain. Sequence diagram = static call order as written in source (guards shown on conditional calls), never a runtime trace.
- (decision) Unit grain: FLOWCHART ONLY in v1. Code view = NATIVE EDITOR SPLIT beside the canvas, selection-synced; the toolkit never re-renders source in a webview.
